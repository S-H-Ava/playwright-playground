# 演習2: 動作確認 - Playwright MCPを使ったスクリーンショットエビデンスの取得

## 概要

この演習では、Playwright MCPを使ってブラウザを実際に操作し、各ステップのスクリーンショットをエビデンスとして取得します。手動テストの証跡をPlaywright MCPで効率的に記録する方法を学びます。

## 学習目標

- Playwright MCPを使ってブラウザ操作とスクリーンショット撮影を組み合わせられる
- エビデンスとして有効なスクリーンショットの撮り方を理解できる
- 動作確認レポートを作成できる
- UI要素の状態変化（カートバッジの更新など）を視覚的に確認できる

## 前提条件

- アプリが起動していること（`npm run dev`）
- Playwright MCPが有効になっていること
- GitHub Copilot Chatが利用できること

---

## 演習手順

### ステップ1: アプリの起動確認

```bash
npm run dev
```

---

### ステップ1.5: カートの初期化（重要）

ブラウザのローカルストレージにカートの状態が残っていると、確認結果が期待値とずれることがあります。
演習を開始する前に、以下のいずれかの方法でカートを空にしてください。

**方法A: カートページから削除する**

`http://localhost:3000/cart` を開き、カートに商品が入っている場合はすべて削除してください。

**方法B: Copilot Chatに依頼する**

```
Playwright MCPを使って http://localhost:3000/cart を開き、
カートに商品が入っている場合はすべて削除してカートを空にしてください。
```

---

### ステップ2: GitHub Copilot Chatに以下のプロンプトを入力

> 💡 **プロンプト例**（コピーして使用してください）

```
Playwright MCPを使って、プレイショップの動作確認を実施してください。
以下の操作をひとつひとつ実行し、各ステップの後にスクリーンショットを撮影してください。

【確認フロー】

1. http://localhost:3000 を開く
   → スクリーンショット撮影（エビデンス1: ホームページ）
   → ヒーローセクションに「プレイショップへようこそ」が表示されていることを確認

2. 「全商品を見る →」ボタンをクリックして商品一覧ページへ移動する
   → スクリーンショット撮影（エビデンス2: 商品一覧）
   → 「10件の商品」と表示されていることを確認

3. 「食品」カテゴリボタンをクリックして絞り込みを行う
   → スクリーンショット撮影（エビデンス3: 食品フィルター後）
   → 「4件の商品」と表示されていることを確認

4. 「有機抹茶パウダー プレミアム」をクリックして商品詳細ページへ移動する
   → スクリーンショット撮影（エビデンス4: 商品詳細ページ）
   → 価格「¥2,400」が表示されていることを確認

5. 「カートに追加」ボタンをクリックする
   → スクリーンショット撮影（エビデンス5: カート追加後）
   → ヘッダーのカートバッジに「1」が表示されていることを確認

6. ヘッダーのカートボタンをクリックしてカートページへ移動する
   → スクリーンショット撮影（エビデンス6: カートページ）
   → カートに商品が表示され、合計金額が正しいことを確認

すべての確認が完了したら、以下の形式で動作確認レポートを作成してください：

| # | 確認項目 | 期待値 | 実際の値 | 結果 |
|---|---------|--------|---------|------|
| 1 | ホームページタイトル | プレイショップへようこそ | [確認した値] | ✅/❌ |
| 2 | 全商品件数 | 10件の商品 | [確認した値] | ✅/❌ |
| 3 | 食品フィルター結果 | 4件の商品 | [確認した値] | ✅/❌ |
| 4 | 有機抹茶パウダーの価格 | ¥2,400 | [確認した値] | ✅/❌ |
| 5 | カートバッジ | 1 | [確認した値] | ✅/❌ |
| 6 | カート合計 | ¥2,400 | [確認した値] | ✅/❌ |
```

### ステップ3: レポートを確認

Copilotが各ステップのスクリーンショットを撮影しながら動作確認を進め、最終的にレポートを生成します。

各エビデンス画像とレポートの内容を確認してください。

## スクリーンショット撮影のポイント

### 全体ページのスクリーンショット

```
browser_screenshot でページ全体のスクリーンショットを撮影してください
```

### 特定要素にフォーカスしたスクリーンショット

```
browser_screenshot で data-testid="cart-badge" の要素にフォーカスして
スクリーンショットを撮影してください
```

## 追加チャレンジ

基本演習が完了したら、以下の確認も試してみてください：

1. **レスポンシブ確認**：ウィンドウサイズを変更して、モバイル表示を確認する

3. **カート操作の確認**：商品を複数追加し、数量変更・削除が正しく動作することを確認する

> ```
