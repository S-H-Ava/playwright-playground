# 演習2: 動作確認 - Playwright MCPを使ったスクリーンショットエビデンスの取得

## 概要

この演習では、Playwright MCPを使ってブラウザを実際に操作し、各ステップのスクリーンショットをエビデンスとして取得します。手動テストの証跡をPlaywright MCPで効率的に記録する方法を学びます。

## 学習目標

- Playwright MCPを使ってブラウザ操作とスクリーンショット撮影を組み合わせられる
- エビデンスとして有効なスクリーンショットの撮り方を理解できる
- 動作確認レポートを作成できる
- UI要素の状態変化（カートバッジの更新など）を視覚的に確認できる

## 前提条件

- アプリが起動していること（`npm run dev`）
- Playwright MCPが有効になっていること
- GitHub Copilot Chatが利用できること

---

## 演習手順

### ステップ1: アプリの起動確認

```bash
npm run dev
```

---

### ステップ1.5: カートの初期化（重要）

ブラウザのローカルストレージにカートの状態が残っていると、確認結果が期待値とずれることがあります。
演習を開始する前に、以下のいずれかの方法でカートを空にしてください。

**方法A: カートページから削除する**

`http://localhost:3000/cart` を開き、カートに商品が入っている場合はすべて削除してください。

**方法B: Copilot Chatに依頼する**

```
Playwright MCPを使って http://localhost:3000/cart を開き、
カートに商品が入っている場合はすべて削除してカートを空にしてください。
```

---

### ステップ2: プロンプトを考えて入力する

GitHub Copilot Chatに、以下の**シナリオを達成するプロンプトを自分で書いて**入力してください。

#### 📋 達成するシナリオ

> プレイショップの以下の流れをブラウザで実際に操作し、各ステップでスクリーンショットを撮影する。
>
> 1. ホームページの表示確認
> 2. 商品一覧で「食品」カテゴリで絞り込み
> 3. 「有機抹茶パウダー プレミアム」の商品詳細ページを確認
> 4. カートに追加してカートページで合計金額を確認
>
> 最後に確認結果を表形式のレポートとしてまとめる。

#### 💡 プロンプトを書くときのヒント

- 各ステップで**スクリーンショットを撮影する**よう依頼する
- 確認したい具体的な**期待値**（件数、価格、バッジの数字など）をプロンプトに含める
- **レポート形式**を指定する（気になる点：期待値、実際の値、結果 ✅/❌）

#### 🔢 このアプリで確認する期待値

| 確認項目               | 期待値                   |
| ---------------------- | ------------------------ |
| ホームページタイトル   | プレイショップへようこそ |
| 全商品件数             | 10件の商品               |
| 食品フィルター結果     | 4件の商品                |
| 有機抹茶パウダーの価格 | ¥2,400                   |
| カートバッジ           | 1                        |
| カート合計             | ¥2,900（送料500円含む）  |

> ❓ プロンプトを書いたら、Copilotが各ステップでスクリーンショットを撮影しながら操作するか確認してください。
> 撮影されないステップがあれば、プロンプトを調整してみましょう。

---

### ステップ3: レポートを確認

Copilotが各ステップのスクリーンショットを撮影しながら動作確認を進め、最終的にレポートを生成します。

各エビデンス画像とレポートの内容を確認してください。

---

## スクリーンショット撮影のポイント

### 全体ページのスクリーンショット

```
browser_screenshot でページ全体のスクリーンショットを撮影してください
```

### 特定要素にフォーカスしたスクリーンショット

```
browser_screenshot で data-testid="cart-badge" の要素にフォーカスして
スクリーンショットを撮影してください
```

## 確認すべき画面状態

### ホームページ（エビデンス1）

- ヒーローセクションのタイトル表示
- おすすめ商品が3件表示されている
- ナビゲーションメニューが表示されている

### 商品一覧（エビデンス2）

- 検索バーが表示されている
- カテゴリフィルターが表示されている
- 「10件の商品」と表示されている
- 商品カードが3列グリッドで表示されている

### 食品フィルター後（エビデンス3）

- 「食品」カテゴリボタンがアクティブ（紺色）になっている
- 「4件の商品」と表示されている
- 食品カテゴリの商品のみが表示されている

### 商品詳細（エビデンス4）

- 商品名「有機抹茶パウダー プレミアム」が表示されている
- 価格「¥2,400」が表示されている
- 在庫状況が表示されている
- レビューセクションが表示されている

### カート追加後（エビデンス5）

- 「✓ カートに追加しました！」と表示されている
- ヘッダーのカートバッジに「1」が表示されている

### カートページ（エビデンス6）

- 追加した商品が表示されている
- 数量「1」が表示されている
- 合計金額「¥2,900」（送料500円含む）が表示されている

## 追加チャレンジ

基本演習が完了したら、以下の確認も試してみてください：

1. **レスポンシブ確認**：ウィンドウサイズを変更して、モバイル表示を確認する
2. **エラー状態の確認**：存在しない商品URLにアクセスして404ページを確認する
3. **カート操作の確認**：商品を複数追加し、数量変更・削除が正しく動作することを確認する

> スクリーンショットをエビデンスとして撮影してください。

> ```
