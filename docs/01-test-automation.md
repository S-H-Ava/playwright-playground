# 演習1: テスト自動化 - Playwright MCPを使ったE2Eテストコードの自動生成

## 概要

この演習では、Playwright MCPを使ってブラウザを実際に操作し、その操作記録をもとにE2Eテストコードを自動生成します。

## 学習目標

- Playwright MCPを使ってブラウザを自動操作できる
- 操作内容を自然言語でCopilotに伝えるプロンプトを書ける
- ページ遷移・クリック・フォーム入力を含むE2Eテストコードを生成できる
- `expect` アサーションで正しい値を検証できる

## 前提条件

- アプリが起動していること（`npm run dev`）
- Playwright MCPが有効になっていること
- GitHub Copilot Chatが利用できること

---

## アプリの概要

| ページ   | URL              | 主な機能                                                         |
| -------- | ---------------- | ---------------------------------------------------------------- |
| ホーム   | `/`              | ヒーローバナー・おすすめ商品3件                                  |
| 商品一覧 | `/products`      | 全10件・カテゴリフィルター（電子機器/衣類/食品）・キーワード検索 |
| 商品詳細 | `/products/{id}` | 商品情報・レビュー・カートに追加                                 |
| カート   | `/cart`          | 商品一覧・数量変更・合計金額（5,000円未満は送料500円）           |

---

## 演習手順

### ステップ1: アプリの起動確認

```bash
npm run dev
```

ブラウザで `http://localhost:3000` を開き、プレイショップが表示されることを確認してください。

---

### ステップ2: プロンプトを入力する

GitHub Copilot Chatに、以下のプロンプトをコピーして貼り付けてください。

#### ✅ プロンプト（コピーして使用してください）

```
## 依頼内容

Playwright MCPでEdgeブラウザを使ってプレイショップのショッピングフローを実際にブラウザで操作し、操作を元にE2Eテストコードを自動生成してください。

`http://localhost:3000` のプレイショップで、以下の操作をPlaywright MCPで実際に実行してください。

1. ホームページを開き、「全商品を見る」ボタンをクリックして商品一覧ページへ移動する
2. 「電子機器」カテゴリで絞り込み、表示件数（3件）を確認する
3. 最初の商品をクリックして商品詳細ページへ移動し、商品名と価格を確認する
4. 「カートに追加」ボタンをクリックする
5. ヘッダーのカートボタンをクリックしてカートページへ移動し、商品が追加されていることと合計金額を確認する

上記の操作が完了したら、この一連の流れをテストコードとして `e2e/shopping-flow.spec.ts` に保存してください。

## テストコードの要件

- `@playwright/test` を使用する
- `test.describe` で「ショッピングフローのテスト」グループを作成する
- 再現性が高く安定して動作するコードを生成する
- 各ステップに適切なアサーションを追加する
- ページ遷移後は遷移完了を待ってから次の操作を行う
- コメントは日本語で記述する
```

> ❓ Copilotの応答を確認してください。意図通りの操作が行われましたか？

---

### ステップ3: 生成されたテストコードを実行する

Copilotが生成したテストコードが `e2e/shopping-flow.spec.ts` に保存されたら、以下で実行してください：

```bash
npm run test:e2e
```

---

### ステップ4: テスト結果を確認する

テストが成功したら、HTMLレポートを確認できます：

```bash
npx playwright show-report
```

> ❓ テストが失敗した場合、どのアサーションが失敗しているか確認し、プロンプトを改善してみましょう。

---
