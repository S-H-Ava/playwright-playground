# 演習3: デバッグ - Playwright MCPを使ったパフォーマンス問題の特定

## 概要

この演習では、デバッグ情報が一切ないアプリに対して、GitHub Copilot と Playwright MCP を駆使して**フロントエンド側のパフォーマンス問題**を調査します。「どこが遅いか」を検証するために、Playwright MCP のブラウザ JavaScript 実行機能や・コードへのデバッグログ追加を活用します。

## 学習目標

- Playwright MCP の `browser_evaluate` でブラウザのJavaScriptを実行して情報収集できる
- `browser_console_messages` でブラウザコンソールのログを取得できる
- Copilot にフロントエンドコードを修正させてデバッグ情報を追加できる
- ネットワーク vs フロントエンドJSのどちらにボトルネックがあるかを判断できる
- パフォーマンス改善の提案ができる

## 前提条件

- アプリが起動していること（`npm run dev`）
- Playwright MCP が有効になっていること
- GitHub Copilot Chat が利用できること

---

## 背景: ダッシュボードの問題

「パフォーマンスダッシュボード」ページ（`/dashboard`）で「レポートを生成」ボタンをクリックすると、**非常に時間がかかる**という報告があります。

ユーザーからのフィードバック：

> 「レポート生成に数秒かかっていて、使いにくい。どこが遅いのか調べてほしい。」

現在のアプリには**パフォーマンスに関するログや計測コードが一切ありません**。「ネットワークが遅いのか、ブラウザ上のJS処理が遅いのか」――画面を見るだけでは判断できません。どんな手段で調査しますか？

## 参考: デバッグに使えるAPI

### Console API（フロントエンドコードへのログ追加）

```javascript
// 処理の実行時間計測（フロントエンドコードに追加する）
console.time("ラベル名");
// ... 計測したい処理 ...
console.timeEnd("ラベル名");
```

### Performance API（ブラウザ側でAPIの通信時間を確認）

```javascript
performance
  .getEntriesByType("resource")
  .filter((entry) => entry.name.includes("/api/"))
  .map((entry) => ({
    url: entry.name,
    duration: Math.round(entry.duration),
    waiting: Math.round(entry.responseStart - entry.startTime),
  }));
```
